# 一、产品概述

## 1.1 产品名称

抖音内容智能 Skill 生成工具（本地版）

## 1.2 产品定位

一个面向技术用户的本地AI工具，将抖音博主视频内容自动转化为可被OpenClaw调用的Skill，实现“内容 → AI能力”的转换。

## 1.3 目标用户

* 技术型用户（程序员）
* OpenClaw 玩家
* AI应用探索者

## 1.4 核心价值

* 将优质短视频内容结构化
* 快速构建“类人格AI”
* 降低AI能力构建门槛

---

# 二、核心功能模块

## 2.1 输入模块

### 支持输入方式

* 博主名称（模糊匹配 + 选择）
* 博主主页链接（推荐）
* 抖音用户ID（精准）

---

## 2.2 视频抓取模块

### 功能

* 获取最近10条视频
* 抓取内容：

  * 视频文件
  * 标题
  * 发布时间
  * 视频链接

### 技术方案

* 基于开源抖音爬虫（如Github方案）
* 模拟浏览器请求
* Cookie机制绕过基础风控

### 存储

* 本地缓存（按博主ID分目录）

---

## 2.3 数据过滤模块（核心）

### 策略：**宁可少，也要纯**

### 过滤规则：

#### 1️⃣ 音频分类

* 纯音乐 ❌（剔除）
* 人声 + 背景音乐 ✅（保留）

#### 2️⃣ 声纹识别

* 判断是否为同一人
* 多人对话 ❌（剔除）

#### 3️⃣ 输出

* 有效视频列表（5~10条）
* 被过滤视频 + 原因（供展示）

---

## 2.4 视频转写模块

### 技术选型

* 推荐：OpenAI Whisper（云端API）

### 功能

* 提取音频
* 转写文本（带时间戳）

### 输出格式

```json
{
  "text": "...",
  "segments": [
    {
      "start": 0.0,
      "end": 3.2,
      "text": "..."
    }
  ]
}
```

---

## 2.5 内容安全检测模块

### 检测维度

* 攻击性语言
* 低俗/违规内容
* 不当价值观

### 技术方案

* LLM审核（GPT）
* 关键词规则补充

### 策略（用户选择：C）

* ⚠️ 标记风险内容
* 不阻断生成

---

## 2.6 AI内容整合模块（核心）

### 输入

* 多条视频转写文本

### 输出

* Skill结构
* 博主画像

---

## 2.7 博主画像生成

### 输出内容

* 内容领域（如：科技 / 情绪 / 商业）
* 表达风格（理性 / 幽默 / 激进）
* 人设标签（如：理性分析型）
* 高频词 / 口头禅
* 目标受众

---

## 2.8 Skill生成模块（关键）

### Skill结构定义（自定义规范）

```json
{
  "name": "Skill名称",
  "description": "描述",
  "persona": {
    "style": "表达风格",
    "tone": "语气",
    "tags": []
  },
  "triggers": ["触发词"],
  "system_prompt": "...",
  "examples": [
    {
      "user": "...",
      "assistant": "..."
    }
  ]
}
```

### 特点

* 模拟语气
* 融合知识
* 包含few-shot示例

---

## 2.9 Skill评分系统

### 评分维度（AI自动打分）

| 维度    | 说明      |
| ----- | ------- |
| 纯度    | 是否单人风格  |
| 可信度   | 是否稳定无胡编 |
| 风格还原度 | 是否像原博主  |
| 可用性   | 是否适合对话  |

### 输出

```json
{
  "score": {
    "purity": 0.9,
    "consistency": 0.85,
    "style": 0.88,
    "usability": 0.92
  }
}
```

---

## 2.10 导出模块

### 支持格式

* JSON（OpenClaw用）
* Markdown（可读）
* Prompt（可直接复制）

---

## 2.11 Skill测试模块

### 功能

* 内置对话窗口
* 与Skill实时对话
* 验证“像不像博主”

---

## 2.12 历史与版本管理

### 功能

* Skill列表
* 版本记录（v1 / v2）
* 可回滚

---

## 2.13 中间结果展示

用户可查看：

* 转写文本
* 被过滤视频
* 参与生成的数据

---

## 2.14 进度系统

### 展示阶段：

* 抓取视频（x/10）
* 转写中（x/10）
* 分析中
* 生成中

---

## 2.15 失败处理机制

### 策略

* 自动降级（10 → 5条）
* 提示原因
* 标记“不适合生成”

OpenClaw 的技能（Skills）核心文件是 **`SKILL.md`**，采用 **YAML Frontmatter + Markdown 正文** 的混合格式，兼容 AgentSkills 规范。

### 一、文件基本结构
```yaml
---
# YAML Frontmatter（元数据区，必须）
name: skill-id           # 唯一ID（小写+连字符）
description: 功能与触发条件 # AI判断调用的核心
version: 1.0.0           # 可选：版本
author: Name             # 可选：作者
user-invocable: true     # 可选：是否支持/命令
disable-model-invocation: false # 可选：禁止AI自动触发
metadata:                # 可选：扩展JSON
  {"openclaw": {"emoji": "🔧"}}
---

# Markdown 正文（技能指令）
## 功能说明
...

## 使用步骤
1. ...
2. ...

## 工具/参数
- 参数1 (类型, 必选): 说明
```

### 二、必填字段
- **`name`**：技能唯一标识（如 `weather`）
- **`description`**：清晰描述功能与触发场景（AI 依赖此字段匹配意图）

### 三、目录结构
每个技能为独立文件夹：
```
skills/
└── my-skill/
    ├── SKILL.md      # 核心定义（必需）
    ├── scripts/      # 可选：脚本
    ├── assets/       # 可选：资源
    └── .env          # 可选：密钥
```

### 四、加载路径（优先级从高到低）
1. `<工作区>/skills/`
2. `~/.openclaw/skills/`
3. 内置技能

### 五、完整示例
```yaml
---
name: file-search
description: 搜索本地文件，支持关键词与类型过滤
version: 1.0.0
author: claw-dev
metadata: {"openclaw": {"emoji": "🔍"}}
---

# 文件搜索技能
## 触发
用户找文件、搜文档、查路径时激活

## 参数
- query (string, 必选): 搜索关键词
- type (string, 可选): 过滤类型（txt/md）

## 命令
```bash
find . -name "*{{query}}*{{type}}"
```
